  <name>Task 2: Run NAMM smoke eval on GPU machine</name>
  <action>
SSH to sideswipe or prowl (NOT blaze — GLIBC incompatible) and run the smoke evaluation. Copy-paste the block below exactly. The eval takes 5-20 minutes; use tmux to protect against SSH disconnection.

STEP 1 — SSH and start tmux:
```bash
# From your local machine:
ssh sideswipe   # or: ssh prowl
tmux new -s namm-smoke-eval
```

STEP 2 — Set up environment:
```bash
source /cs/student/project_msc/2025/csml/gmaralla/miniconda3/etc/profile.d/conda.sh
conda activate th2
export HF_HOME="/cs/student/project_msc/2025/csml/gmaralla/.hf_cache"
cd /cs/student/project_msc/2025/csml/gmaralla/NAMM_implementation
```

STEP 3 — Verify you are on the correct branch:
```bash
git branch    # must show: * dev/joint-namm-lora-es
git status    # must show: nothing to commit, working tree clean
```

STEP 4 — Run the smoke eval:
```bash
torchrun --standalone --nproc_per_node=1 main.py \
    'run@_global_=namm_bam_eval_llama32_1b.yaml' \
    init_from=/cs/student/project_msc/2025/csml/gmaralla/NAMM_implementation/exp_local/memory_evolution_hf/Llama-3.2-1B/NAMM/attn-spec-norm/bam/binary-1024cs/qasper-cma-es-p8-rMeanTrue-shared-8pop-16qs-256fixDel-llama32-1b-stage1/1337/ckpt.pt \
    cache_size=128
```

IMPORTANT: Single-quotes around `'run@_global_=namm_bam_eval_llama32_1b.yaml'` are required. If running from csh instead of bash, wrap the entire torchrun call in `bash -c '...'`.

STEP 5 — Record the QASPER score. When the eval completes, look for a line like:
```
val_lb/qasper: 6.XX
```
Or extract from the most recent eval JSON:
```bash
python3 -c "
import json, glob
files = sorted(glob.glob('exp_local/**/eval_*.json', recursive=True))
if files:
    with open(files[-1]) as f:
        d = json.load(f)
    print('QASPER:', d.get('val_lb/qasper', d.get('train_lb/qasper', 'NOT FOUND')))
"
```

PASS CRITERIA: QASPER F1 in range [6.36, 6.56].
- Score outside this range but above 6.0: run once more to check for noise before diagnosing.
- Score significantly above 6.46 (e.g. ~7.0): cache_size was not applied — re-check the torchrun command.
- Score below 6.0: wrong checkpoint or config — report before proceeding.
